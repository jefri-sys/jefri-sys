## Engineering Philosophy

```
Software should solve a problem that actually exists — not an abstraction of one.
```

Three principles shape every system I design:

**Clarity over cleverness.** Code is read far more often than it is written. I optimize for the engineer who inherits the codebase at 2 AM — that engineer is often future me.

**AI as infrastructure, not a feature.** Generative AI belongs in the architecture layer — embedded in pipelines, callable by services, monitored like any other system component. Bolting it on top creates fragile demos, not products.

**Ship, then sharpen.** A working system with known trade-offs beats a perfect design that never ships. I document constraints, make trade-offs explicit, and iterate with real usage data.

<summary><strong>🧠 Synapse — Enterprise AI-Powered Student Productivity Platform</strong></summary>
<br/>

> **Lead Engineer · Full Stack · AI Systems**

### Overview

Synapse is a multi-module SaaS platform designed to consolidate the fragmented tooling that students use across productivity, academics, and career development — into a single AI-augmented workspace.

### Problem Statement

Students operate across a dozen disconnected tools: note apps, job boards, attendance trackers, expense managers, and chat platforms. There's no contextual AI layer tying these together. Synapse fills that gap.

### Solution

A unified MERN-stack platform where an AI layer (RAG pipeline + Gemini API) has access to the user's own notes, documents, and career data — enabling contextual answers, resume analysis, and intelligent task prioritization across all modules.

### Core Modules

| Module | Description |
|---|---|
| **AI Notebook** | Write notes, tag content, and query your own knowledge base via a RAG pipeline |
| **Career Vault** | Track applications, store JD data, map skills to job requirements |
| **Resume Intelligence** | Upload resumes; AI identifies gaps, suggests improvements, scores ATS fit |
| **Attendance Tracker** | Mark, visualize, and project attendance percentage with smart alerts |
| **Expense Tracker** | Log and categorize expenses; visualize spending patterns |
| **Task Manager** | Priority-based task lists with deadlines and completion tracking |
| **Smart Notes** | Rich-text notes with search, tagging, and AI summarization |
| **Realtime Chat** | One-to-one and group chat with Socket.io |
| **Groups** | Project-based collaboration spaces with shared files |

### Architecture Highlights

- **RAG Pipeline**: Document chunks stored as vector embeddings; Gemini handles retrieval-augmented generation
- **Multi-LLM Orchestration**: Fallback logic across Gemini model tiers to manage quota constraints
- **JWT Auth**: Secure stateless authentication with refresh token rotation
- **Progressive Web App**: Installable, offline-capable, push notification support
- **Cloudinary**: Media pipeline for document and image storage with CDN delivery
- **Socket.io**: Bidirectional real-time events for chat, notifications, and collaboration

<summary><strong>🖨️ SCRMS — Smart Campus Reprography Management System</strong></summary>
<br/>

> **Solo Engineer · Full Stack · Production Deployment**

### Overview

SCRMS is a campus-scale digital printing management platform that replaces manual print shop workflows with a real-time, role-based system — covering order placement, queue management, payment, and admin analytics.

### Problem Statement

Campus print shops operate entirely on paper queues and manual cash transactions. Students wait without visibility into order status. Staff have no data on inventory consumption or demand patterns. SCRMS eliminates this entirely.

### Solution

A production-deployed MERN-stack PWA with role-based dashboards for students, staff, and admins. Orders flow through a real-time queue system with live status updates, automated payment via Razorpay, and a wallet system with atomic MongoDB transactions.

### Core Features

| Feature | Implementation |
|---|---|
| **Role-based Auth** | Student · Staff · Admin dashboards with JWT-protected routes |
| **Order Management** | Print · Photocopy · Scan · Bind — full lifecycle tracking |
| **Real-time Queue** | Socket.io live order status pushed to student and staff screens |
| **OTP Verification** | Email OTP on registration; receipt OTP on physical pickup |
| **Payment System** | Razorpay gateway + in-app wallet with atomic MongoDB transactions |
| **Group Split Pay** | GPay-style payment splitting in group chat — auto settlement |
| **Admin Analytics** | Revenue, volume, service type breakdowns, inventory consumption |
| **One-to-One Chat** | Student–staff messaging with media and voice note support |
| **Group Chat** | Group rooms with file sharing and integrated split payment |
| **Gemini AI Chatbot** | Context-aware assistant for order help and campus queries |
| **DocConvert Pipeline** | CloudConvert API integration for .docx-to-PDF rendering |
| **Push Notifications** | PWA web push — order status, payment, and chat alerts |
| **Receipt Generation** | PDF receipts generated server-side after payment confirmation |
| **Complaint Management** | Structured complaint flow with staff escalation and resolution tracking |
| **Inventory Tracking** | Paper, ink, and consumable stock with low-stock alerts |

### Architecture Highlights

- **Atomic Wallet Transactions**: MongoDB sessions to prevent double-spend on concurrent wallet debits
- **Integration Testing**: 117-test suite covering API contracts; known failures documented rather than suppressed
- **Cloud Deployment**: Frontend on Vercel, backend on Railway, media on Cloudinary, DB on MongoDB Atlas
- **Offline-capable PWA**: Service workers handle asset caching; push API handles server-side notifications
